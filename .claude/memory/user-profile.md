- **Background**: ECE student. Has CS and webdev subskills from coursework/projects but lacks depth in principles (API design, DB normalization, system design patterns).
- **Experience level**: Intermediate in practice — can read and write code, has built things (personal projects). Weak on the *why* behind patterns and conventions.
- **Goal**: This project exists to build that missing foundation. Learning by doing a CRUD experiment across REST and GraphQL.
- **Learning style**: Wants to understand concepts, not just copy-paste. Prefers guided Q&A walkthroughs. Ask "why does this work this way?" type questions — answer them proactively.

**How to apply:** Surface the mental model, not just the syntax. Frame explanations around principles (e.g. why REST uses nouns in URLs, why GraphQL has a schema). Assume code syntax is familiar; assume design rationale is not.
