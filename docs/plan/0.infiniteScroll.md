# Plan: Infinite Scroll for Market + Order History

## Context

หน้า Market ([ingredients.page.jsx](vscode-webview://1na62d8gf4jeoqto5g3q3a4autsg18hnmqpd63ne30atd2mg8o30/frontend/src/pages/ingredients.page.jsx)) และ Order History ([orderHistory.page.jsx](vscode-webview://1na62d8gf4jeoqto5g3q3a4autsg18hnmqpd63ne30atd2mg8o30/frontend/src/pages/orderHistory.page.jsx)) ตอนนี้ fetch ครั้งเดียวโหลดทั้งก้อน — ไม่มี pagination. Backend พร้อมแล้ว (offset-based, `?page=&limit=`, response `{ total, page, limit, data }`) ทั้ง list + search endpoints. เป้าหมาย: infinite scroll ทั้ง 2 หน้า ด้วย shared hook, และย้าย filter/search ของ Market ไป server-side (ไม่งั้น client filter จะกรองแค่หน้าที่โหลดมา).

## Backend contract (ยืนยันแล้ว)

|Endpoint|Path|Auth|Params|Response|
|---|---|---|---|---|
|Ingredient list|`/api/ingredients`|public|`page`, `limit` (def 15)|`{ total, page, limit, data }`|
|Ingredient search|`/api/ingredients/search`|public|`q`, `category`, `inStock`, `page`, `limit`|same|
|Order list|`/api/order`|JWT|`page`, `limit` (def 20)|same|
|Order search|`/api/order/search`|JWT|`value` (required), `page`, `limit`|same|

`hasMore` = `loadedCount < total`.

## Change 1 — NEW `frontend/src/hooks/useInfiniteList.js`

Generic hook. ไม่รู้จัก endpoint — page ส่ง `fetchPage(page) → { data, total }` เข้ามา.

Signature:

```js
useInfiniteList(fetchPage, { limit = 15, enabled = true })
  → { data, loading, hasMore, sentinelRef }
```

รับผิดชอบ:

- state: `data[]`, `page`, `total`, `loading`
- **reset**: `useEffect([fetchPage, enabled])` → เมื่อ `fetchPage` identity เปลี่ยน (query/category เปลี่ยน) refetch page 1, แทน data. ใช้ `cancelled` flag กัน race (last-write-wins).
- **loadMore**: `page+1`, append `[...prev, ...res.data]`, guard `loading || !hasMore`.
- **observer**: `IntersectionObserver` บน `sentinelRef`, root = viewport; fire `loadMore` เมื่อ intersecting; `disconnect` ใน cleanup.
- `enabled=false` (config/token ยังไม่พร้อม) → effect ไม่ทำงาน.

## Change 2 — [ingredients.page.jsx](vscode-webview://1na62d8gf4jeoqto5g3q3a4autsg18hnmqpd63ne30atd2mg8o30/frontend/src/pages/ingredients.page.jsx)

- ลบ `fetchItems`, `items` state, `visible` useMemo (client filter ออกทั้งหมด).
- เพิ่ม `fetchPage` useCallback (deps `[config, query, category, addLog]`):
    - endpoint: มี `query||category` → `API_ENDPOINT_INGREDIENT_SEARCH`, ไม่มี → `API_ENDPOINT_INGREDIENT`
    - `URLSearchParams({ page, limit:15, ...(query&&{q:query}), ...(category&&{category}) })`
    - return `{ data: res.data ?? [], total: res.total ?? 0 }`
- `const { data: items, loading, hasMore, sentinelRef } = useInfiniteList(fetchPage, { limit:15, enabled: !!config })`
- render `items.map` (แทน `visible.map`) — grid เดิม ไม่แตะ card markup.
- เพิ่ม `<div ref={sentinelRef} />` ใต้ grid + spinner เมื่อ `loading && hasMore`.
- SearchBar/CategoryBar เดิมคงไว้ — ตอนนี้ drive server query ผ่าน `fetchPage` deps.

## Change 3 — [orderHistory.page.jsx](vscode-webview://1na62d8gf4jeoqto5g3q3a4autsg18hnmqpd63ne30atd2mg8o30/frontend/src/pages/orderHistory.page.jsx)

- ลบ `fetchOrders`, `orders` state, manual `useEffect`.
- เพิ่ม `fetchPage` useCallback (deps `[config, query, user?.accessToken, addLog]`):
    - endpoint: `query` → `API_ENDPOINT_ORDER_SEARCH` (param `value`), ไม่มี → `API_ENDPOINT_ORDER`
    - `URLSearchParams({ page, limit:20, ...(query&&{value:query}) })`
    - token = `user.accessToken`
- `const { data: orders, loading, hasMore, sentinelRef } = useInfiniteList(fetchPage, { limit:20, enabled: !!config && !!user?.accessToken })`
- render table เดิม + `openOrder`/`selectedOrder` box ไม่แตะ.
- เพิ่ม `<tr ref={sentinelRef} />` หรือ sentinel div ใต้ table + spinner เมื่อ `loading && hasMore`.

## Notes / ponytail

- **ไม่ debounce search** — match พฤติกรรม per-keystroke เดิม. seed data เล็ก. ถ้า catalog โต ค่อยเพิ่ม debounce 300ms ใน hook (mark `// ponytail: no debounce, add if server calls hurt`).
- ตัด `inStock` filter ออกก่อน — UI ยังไม่มีปุ่มนั้น.
